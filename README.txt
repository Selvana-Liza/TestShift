Параметры программы задаются при запуске через аргументы командной строки, по порядку:
1. режим сортировки (-a или -d), по умолчанию сортируем по возрастанию (необязательный);
2. тип данных (-s или -i) (обязательный);
3. имя выходного файла (обязательный);
4. остальные параметры – имена входных файлов, минимум 2 (обязательный).

Пример запуска:
java Main -i out.txt in1.txt in2.txt
java Main -i -a out.txt in1.txt in2.txt
java Main -s -d out.txt in1.txt in2.txt in3.txt

Проблема считывания больших файлов решается путем разбиения файлов на более мелкие временные текстовые файлы.
По умолчанию стоит разбиение на файлы по 2000 строк.
Берутся 2 файла, разбиваются на n временных файлов, сортируются, если есть еще входные файлы,
результирующий файл полученный слиянием предыдущих двух файлов и новый файл снова разбиваются и т.д.

Поврежденные файлы (нарушение сортировки и наличие пробеллов) проверяется при считывании строк.
Все строки выбивающиеся из сортировки (наличие пробела) игнорируются. 
Т.е. ечли у нас есть файл {1,2,3,8,4,9}, то при считывании 4, программа ее проигнорирует и перейдет к 9,
и будет работать с массивом {1,2,3,8,9} 

Файлы для проверки
in1.txt,in2.txt,in3.txt - целочисленные по возрастанию
in11.txt,in22.txt - целочисленные по убыванию
in4.txt,in5.txt,in6.txt,in7.txt - строковые по возрастанию
in9.txt,in10.txt - строковый по убыванию
in8.txt - смешанный
